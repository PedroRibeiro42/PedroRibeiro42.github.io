SSSP (single source shortest path problem)

1) Implemente e submeta uma solução para o problema [DAA 027] - Viagem para as Aulas Teóricas.

Dicas:

  • A sua primeira tarefa é atribuir um número a cada nome de um nó. No final da leitura o grafo construido deve ser entre "números" e não entre "strings". Pode por exemplo fazer uma função numero que receba uma string e devolva um inteiro. Para isso precisa de um array auxiliar que guarde os nomes. Para saber o número, procure nesse array. Caso não exista o nome, adicione e devolva o novo número. Outra alternativa é usar um map que mapeia strings em inteiros.
  • Para descobrir a distância mínima de um ponto a outro num grafo pesado podemos usar o algoritmo de Dijkstra [ver slides 4 a 8]
  • Como o |V| máximo é relativamente pequeno (1000) podemos usar uma versão "naive" do algoritmo de Disjktra (um ciclo para descobrir o mínimo, ou seja, um {\tt choose\_best} em \mathcal{O}(|V|)), que resulta numa complexidade final de \mathcal{O}(|V|^2).
  • Se no final se quiser testar, experimente passar a usar heaps, para atingir uma complexidade melhor (tal como fez com o algoritmo de Prim na aula de árvores de suporte de custo mínimo).

SSSP (single source shortest path problem)

2) Implemente e submeta uma solução para o problema [DAA 028] - Buracos Negros.

Dicas:

  • Este problema pede para descobrir se existem ciclos negativos
  • Para descobrir a distâncias mínimas e saber se existem ciclos de soma negativa podemos usar o algoritmo de Bellman-Ford, que tem uma complexidade temporal de \mathcal{O}(|V| \cdot |E|) se usarmos simplesmente uma lista das arestas [ver slides 11 a 15]

Fecho Transitivo

3) Implemente e submeta uma solução para o problema [DAA 029] - Ligações Aéreas.

Dicas:

  • O que o problema pede chama-se de fecho transitivo de um grafo.
  • Como o |V| máximo é pequeno (26) podemos usar o algoritmo de Floyd-Warshall, que é \mathcal{O}(|V|^3) [ver slides 17 a 25]
